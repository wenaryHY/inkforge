# InkForge 架构全景决策文档

> 生成时间：2026-04-09
> 来源：无尽审问式头脑风暴，共 14 轮假设，35 项核心决策

---

## 一、项目定位与分发

| # | 决策项 | 结论 |
|---|--------|------|
| 1 | 定位 | 双受众：普通用户（桌面版开箱即用）+ 开发者（主题/插件生态） |
| 2 | 分发 | Tauri 桌面版为主力分发形式 |
| 3 | 安装向导 | Tauri 原生引导 UI + Web 安装向导，逻辑共享 |

---

## 二、存储与同步

| # | 决策项 | 结论 |
|---|--------|------|
| 4 | 存储 | SQLite + 完全离线优先 |
| 5 | 同步模型 | 多端发布：InkForge Server 适配器直推（保留动态功能）+ 静态构建管线（纯 HTML 输出），用户配置目标类型自动选策略 |
| 6 | 同步实现 | 先自建 InkForge Server，远期官方 Cloud |
| 7 | 备份 | 本地自动定时备份 + 手动导出远程备份 |

---

## 三、插件系统

| # | 决策项 | 结论 |
|---|--------|------|
| 8 | 插件架构 | 双轨：WASM（FlatBuffer 共享内存，高性能）+ JS sidecar 独立进程（按需启动，低门槛） |
| 9 | 插件 UI | Slot 注入（小部件）+ Route 注册（独立页面），Web Component 桥接，开发者自选前台(3000)/后台(5173) |
| 10 | 插件权限 | 分级信任：官方全权限 / 社区受限 / 手动安装需授权 |
| 11 | 插件配置 | Schema 驱动自动表单 + 自定义页面覆盖 |
| 12 | 插件间通信 | 事件总线（松耦合通知）+ 服务注册（按需直接调用） |
| 13 | 插件安全 | 自动化代码扫描 + 风险标签，不阻塞发布，提供透明度 |
| 14 | 插件沙箱 | 每个插件独立数据目录 + 权限声明访问额外路径 |

---

## 四、主题系统

| # | 决策项 | 结论 |
|---|--------|------|
| 15 | 主题开发 | 构建模式：开发者用任意框架开发，`ink build` 输出纯 HTML/CSS/JS |
| 16 | 主题-插件关系 | 主题可声明依赖插件，缺少时弹窗引导安装 |
| 17 | 模板引擎 | 保持 MiniJinja，未来按需加 ink: 属性预处理器（不需要 IR 中间层） |
| 18 | 主题渲染统一 | MiniJinja 主题视为一种"构建模式"（构建步骤=MiniJinja渲染），统一加载逻辑 |
| 19 | CSS 变量 | 主题完全自主，核心不提供变量系统 |
| 20 | 主题市场 | 先不做商业，架构预留 |
| 21 | SEO | `ink build` 时自动预渲染所有页面为静态 HTML，搜索引擎直接抓取 |

---

## 五、数据注入与 API

| # | 决策项 | 结论 |
|---|--------|------|
| 22 | 数据注入 | SSR JSON 注入 `<script type="application/json" id="ink-data">` + CSR API 增强 |
| 23 | API 策略 | REST 为主 + SDK 封装（JS/TS + Rust），URL 前缀版本 `/api/v1/` |
| 24 | 主题构建产物 | 静态 HTML + API 调用（首屏数据注入 + 动态交互调 API） |
| 25 | ink build | 封装底层构建工具（跑 package.json 的 build 脚本），统一入口 |

## 六、内容能力

| # | 决策项 | 结论 |
|---|--------|------|
| 26 | 内容类型 | 文章 + 页面 + 作品集（type 字段区分，共享表结构） |
| 27 | 修订历史 | 主表 + 修订表（完整快照），用 similar 库做差异对比展示 |
| 28 | 搜索 | 迁移到 tantivy |
| 29 | 权限模型 | NGAC（设计时预留图结构数据模型，避免后期重构） |
| 30 | 编辑器 | 切换到 Tiptap（MIT 许可，Headless，基于 ProseMirror），立即替换 Milkdown |
| 31 | 编辑器迁移 | 最小可用优先：先实现基础 Markdown 编辑，逐步加高级功能 |

## 八、开发者工具链与生态

| # | 决策项 | 结论 |
|---|--------|------|
| 36 | CLI | `ink` 胶水层：dev / create / build / test / publish |
| 37 | 包管理 | 不造新包管理器，前端依赖用 bun，WASM 依赖用 cargo，ink 管生态发布 |
| 38 | Registry | 渐进式：v0.x 用静态 JSON index（GitHub），生态成熟后上官方 API 服务 |
| 39 | 遥测 | 匿名使用统计 + 可选崩溃报告 |
