## Anthropic 官方资源集成策略

### 资源类型

#### 1. Claude Code Skills

**定义**：Anthropic 官方提供的代码技能库，包含最佳实践和模式。

**获取方式**：
- Claude 官方文档
- GitHub 官方仓库

**集成方式**：
- 在 memories 中创建对应的 skill 文件

- 创建 `memories/patterns/` 目录

每个 skill 文件应包含以下内容：

```markdown
# [Skill 名称]

## 概述
- 简要说明
- 适用场景
- 学习时间估计

## 核心概念
- 基础概念 1
- 基础概念 2
- ...

## 最佳实践
- 实践 1
- 实践 2
- ...

## 常见模式
- 模式 1：描述 + 代码示例
- 模式 2：描述 + 代码示例
- ...

## 项目应用
- 当前项目中的应用
- 改进建议
- 相关代码位置

## 参考资源
- 官方文档链接
- 推荐阅读
- 相关工具

## 检查清单
- [ ] 理解核心概念
- [ ] 掌握最佳实践
- [ ] 能应用到项目
- [ ] 代码审查通过
```

## 常见问题

### Q: 如何判断一个资源是否值得引入？

A: 使用"资源评估清单"进行评估。优先选择官方或社区认可的资源，避免过时或低质量内容。

### Q: Skill 文件应该多详细？

A: 应该包含核心概念、最佳实践、常见模式和项目应用。长度通常在 500-2000 字之间。

### Q: 如何保持 skills 的更新？

A: 建立定期审查机制（每月一次），检查链接有效性和内容准确性。

### Q: 新成员如何学习这些 skills？

A: 根据项目需求，推荐相关的 skill 文件。可以在 code review 时引用相关 skills。
